# WebRop 使用指南

## 一、环境要求

- **Node.js** >= 18
- **npm** >= 9
- **浏览器**：Chrome / Edge / Firefox（推荐使用最新版）

## 二、启动项目

```bash
# 1. 进入项目目录
cd WebRop

# 2. 安装依赖（首次运行或依赖有变更时）
npm install

# 3. 启动开发服务器
npm run dev
```

启动后终端会显示 `http://localhost:3000`，在浏览器打开即可。

## 三、连接方式

项目提供两种运行模式：

### 模式 A：Mock 模式（离线模拟，无需 ROS）

适合演示和调试，浏览器内自包含模拟环境。

**操作步骤：**
1. 打开页面后，点击侧边栏 **"Mock Mode"** 按钮
2. 页面自动加载一张默认地图（带墙壁障碍物），机器人出现在地图中
3. 所有功能在浏览器内模拟运行，不依赖外部 ROS 环境

**预期效果：**
- 地图显示黑色障碍物和白色可通行区域
- 一个蓝色带旋转激光雷达的 3D 机器人出现在地图中
- 右上角小地图显示俯视图
- 侧边栏出现 **Mock Log**（模拟事件日志）
- 状态栏显示紫色 "(mock)" 标识

### 模式 B：ROS 模式（连接真实机器人）

需要外部运行 ROS 1 + rosbridge_server。

**操作步骤：**
1. 确认环境中已运行 rosbridge_server：
   ```bash
   roslaunch rosbridge_server rosbridge_websocket.launch
   ```
2. 页面中输入 rosbridge WebSocket 地址（默认 `ws://localhost:9090`）
3. 点击 **"Connect"** 按钮
4. 状态栏变为绿色"ROS Connected"

**预期效果：**
- 状态栏连接指示灯变绿
- 自动接收 `/map`（占用栅格地图）、`/odom`（机器人位姿）、`/particlecloud`（AMCL 粒子云）
- 3D 场景显示真实地图和机器人位置
- 右上角小地图更新为真实地图数据

## 五、操作模式详解

在侧边栏顶部的 **Mode Selector** 切换六种操作模式。

### 5.1 Navigate（导航模式）

**用途：** 给机器人设定移动目标点，让机器人自己导航过去。

**操作步骤：**
1. 确保处于 Mock 模式 或 已连接 ROS
2. Mode Selector 选择 **Navigate**
3. 在地图上点击 → 出现编号 **①** 的航点标记（粉色环）
4. 继续点击添加更多航点：**②**、**③** 等
5. 侧边栏 **Waypoints** 区域会列出所有航点
6. 点击 **"Start Navigation"** 开始导航
7. 机器人按顺序走向每个航点

**预期效果（Mock 模式）：**
- 机器人沿规划路径（白色虚线）移动到第一个航点
- 到达后自动转向下一个航点
- 当前执行中的航点标记高亮（粉色）
- 状态栏显示速度 V/W 值
- 橙色面包屑轨迹跟随机器人运动

**预期效果（ROS 模式）：**
- 发布导航目标到 `/move_base_simple/goal`
- 订阅到的 move_base 规划路径以白色虚线显示
- 机器人按真实导航系统移动

**航点属性编辑：**
- 点击航点 → 选中后在侧边栏显示属性面板
- 可设置每个航点的 **Speed**（到达速度）、**Wait**（到达后等待时间）、**Yaw**（目标朝向）
- 3D 场景中航点会显示紫色箭头指示目标朝向
- 拖拽航点（超过 4px 阈值）可调整位置，短按为选中

**额外功能（侧边栏）：**
- **Fleet**：添加多台机器人，选择机器人类型（car/humanoid/drone/dog），切换激活的机器人
- **Formation**：选择编队类型（直线/纵列/V形/圆形），调整间距
- **Pose Sync**：位姿同步启停
- **Labels**：可拖动滑块恢复已保存的标签

### 5.2 Relocate（重定位模式）

**用途：** 设置机器人初始位姿（2D Pose Estimate），纠正定位偏差。

**操作步骤：**
1. Mode Selector 选择 **📍 Relocate**
2. 在地图上点击 → 设置机器人位置
3. 按住拖拽 → 拖拽方向决定机器人朝向（箭头预览）
4. 释放鼠标 → 应用位姿

**预期效果：**
- 3D 场景中显示位姿预览（箭头 + 圆环 + 标签）
- Mock 模式：直接更新模拟机器人位姿
- ROS 模式：发布 `/initialpose`（PoseWithCovarianceStamped）

**AMCL 定位质量指示（状态栏）：**
- 📡 **Good**（绿色）：定位正常
- 📡 **Fair**（黄色）：定位一般
- 📡 **Poor**（红色）：定位偏差大，建议重定位
- 括号内数字为粒子数量

**Mock 模式额外功能：**
- **Simulate Drift** 滑块：手动模拟 AMCL 定位漂移
- 3D 场景中显示彩色粒子云（绿=好/黄=一般/红=差）
- 定位质量差时每 30 秒弹出警告提示

### 5.3 Map Edit（地图编辑模式，仅 Mock 模式）

**用途：** 修改模拟地图，绘制/擦除障碍物。

**操作步骤：**
1. 先进入 **Mock Mode**
2. Mode Selector 选择 **Map Edit**
3. 侧边栏 Action Panel 显示编辑工具：
   - **Wall**（画墙）— 在地图上拖拽绘制黑色障碍物
   - **Erase**（擦除）— 拖拽擦除已画的障碍物
   - **Rectangle**（矩形）— 点击拖拽绘制矩形障碍区
   - **Place Robot**（放置机器人）— 点击将机器人移到指定位置
4. 通过 **Brush Size** 滑块调整笔刷大小（1~15）
5. 点击 **Reset Default Map** 恢复默认地图
6. 点击 **Clear All Walls** 清除所有墙壁（只保留边界）

**预期效果：**
- 拖拽时地图上出现圆形预览光标
- 释放鼠标后地图黑色区域更新
- 机器人自动避开新画的障碍物（导航时）
- 可以将机器人移动到任意空地

**地图编辑撤销：**
- 地图编辑拥有**独立的撤销栈**（最多 60 步），与区域/路径编辑互不影响
- **Ctrl+Z**：撤销上一次编辑操作（每笔/每矩形/每次重置/每次清空各为一步）
- **Ctrl+Y**：重做
- 状态栏 Ctrl+Z/Y 指示器在地图编辑模式下显示地图撤销状态

### 5.4 HRZ Zone（禁区编辑模式）

**用途：** 在地图上画多边形区域，标记机器人不能去或需要慢速通过的地方。

**操作步骤：**
1. Mode Selector 选择 **HRZ Zone**
2. 在地图上点击放置多边形顶点（白色小圆球）
3. 放完 3 个以上顶点后，靠近第一个顶点（黄色大球）点击 → 多边形自动闭合
4. 侧边栏出现刚刚画好的区域，默认类型为 **Forbidden**（红色）
5. 可在侧边栏切换区域类型：
   - 🔴 **Forbidden**（禁止）— 机器人不可进入，代价=254
   - 🟡 **Slow**（慢速）— 进入该区域时减速，代价=128
   - 🟢 **Charging**（充电）— 充电区域，代价=0

**预期效果：**
- 闭合后显示半透明多边形（带呼吸闪烁动画和区域轮廓线）
- 侧边栏区域列表显示每个区域名称、类型、面积
- 区域名称可编辑
- 已闭合的区域可点击顶点拖拽微调形状
- 进入该区域的路径段自动变红（不可达）或变黄（慢速）
- 区域内显示类型标签文字

**侧边栏按钮：**
- **Apply / Publish Zones** — Mock 模式写入模拟地图 / ROS 模式发布到 `/hrz_zones`
- **Cancel Drawing** — 取消当前正在画的区域
- **Clear All** — 清除所有区域（可 Ctrl+Z 撤销）

**撤销：** Ctrl+Z / Ctrl+Y 撤销/重做区域编辑操作。

### 5.5 HRP Path（路径编辑模式）

**用途：** 手绘机器人导航参考路径，每段可设置不同速度。

**操作步骤：**
1. Mode Selector 选择 **HRP Path**
2. 在地图上点击拖拽绘制路径
3. 路径显示为绿色线条，每段颜色根据速度渐变（绿=快，黄=慢）
4. 侧边栏显示各段速度滑块，可分别调节
5. 右击路径上任意位置 → 在点击处插入新的中间点
6. 拖拽路径上的小白点 → 微调路径形状

**侧边栏操作：**
- **Segment Speed**：13 档速度（0.05~2.0 m/s），每段独立设置
- **Auto Speed (Zone Match)**：根据路径与 HRZ 区域的重叠自动匹配各段速度
- **Check Reachability**：检查路径是否穿过障碍物，阻塞段变红色虚线
- **Follow Path**（Mock 模式）/ **Publish Path**（ROS 模式）

**预期效果：**
- 点击拖拽后出现绿色路径线条
- 每段颜色反映速度（由绿到黄渐变）
- 速度数值标签显示在路径段中点位置
- 侧边栏显示 **Distance**（总距离）和 **Est. Time**（预估通行时间）
- 路径起点有一条虚线连接到机器人位置（绿色=近，黄色=远）

**撤销：** Ctrl+Z / Ctrl+Y 撤销/重做路径编辑操作。

### 5.6 Tasks（任务编排模式）

**用途：** 创建、调度和执行自动化任务链，支持定时触发和条件触发。

**操作步骤：**
1. Mode Selector 选择 **📋 Tasks**
2. 侧边栏显示任务面板，包含四个标签页：

**🔗 Chains（任务链）：**
- 点击 **"+ Chain"** 创建空任务链
- 可添加三种步骤类型：
  - **Path**：引用当前 HRP 路径，机器人沿路径行走
  - **Waypoint**：导航到指定坐标，可在侧边栏编辑 X/Z 或在地图上点击设置
  - **Wait**：原地等待指定秒数
- 可拖拽步骤排序
- 点击 **▶** 执行任务链，**⏹** 取消执行
- 执行进度实时显示在步骤列表中

**⏰ Schedule（定时任务）：**
- 选择已有任务链 + 机器人 + Cron 表达式
- 系统自动计算下次执行时间
- 支持启用/禁用

**⚡ Triggers（条件触发）：**
- 三种条件类型：
  - **battery_low**：电量低于阈值时触发
  - **battery_critical**：电量极低时触发
  - **idle_timeout**：机器人空闲超过指定时间后触发
- 触发后自动执行关联的任务链

**📋 Board（执行面板）：**
- 显示所有任务链的当前执行状态
- 状态：pending / running / completed / failed / cancelled
- 可在此启动或取消任务

## 七、高级功能

### 多机器人车队（Fleet）

**位置：** Navigate 模式侧边栏 **Fleet** 区域

1. 点击 **"+ Robot"** 添加新机器人
2. 选择机器人类型：🚗 Car / 🤖 Humanoid / 🚁 Drone / 🐕 Dog
3. 每种类型有不同的 3D 模型外观
4. 点击机器人名称切换激活的机器人
5. 所有航点和导航操作针对当前激活的机器人
6. 电池模拟：自动缓慢耗电，可通过 Action Panel 的电池滑块手动调节

### 编队（Formation）

**位置：** Navigate 模式侧边栏 **Formation** 区域

- **Line**（直线）/ **Column**（纵列）/ **V-Shape**（V 形）/ **Circle**（圆形）
- 调整间距滑块控制编队密度

### 场景快照

**用途：** 保存当前编辑状态，后续可一键恢复。

**操作位置：** 侧边栏中部 **Snapshots**

1. 在文本框中输入快照名称
2. 点击 **"Save"** 保存当前状态（包含：HRZ 区域、HRP 路径、速度数据、3D 标签、相机位置）
3. 点击已保存的快照名称 → 一键恢复场景

### 膨胀层可视化

**位置：** 状态栏中的 **Inflation** 按钮

点击切换显示：
- 红色区域 = 障碍物本身（占用栅格）
- 橙色区域 = 机器人膨胀范围（距障碍物 8 个栅格单位，机器人半径 0.16m）

**预期效果：** 帮助判断路径是否离障碍物太近。

### 地图覆盖对比

**位置：** 状态栏中的 **Old Map** 滑块

当地图被编辑后，旧地图自动保存为覆盖层：
- 拖拽滑块调节旧地图的透明度（0~100%）
- 方便对比编辑前后的地图变化

### WASD 遥控

**位置：** 状态栏中的 **WASD** 按钮

1. 点击开启遥控模式
2. 使用 W/A/S/D 键控制机器人移动
3. ROS 模式：发布 `/cmd_vel` 到真实机器人
4. Mock 模式：控制模拟机器人

### 3D 标签

**操作：** 在 3D 场景中双击任意位置 → 添加文字标签

**管理位置：** 侧边栏 Action Panel（Navigate 模式下的 Labels 区域）
- 侧边栏显示所有标签列表
- 可删除单个标签或一键清空
- 标签自动持久化

### 路径可达性预检

**位置：** HRP 模式下的 **"Check Reachability"** 按钮

1. 画完路径后点击该按钮
2. 系统检查路径各段是否穿过障碍物
3. 阻塞的路径段变为红色虚线并闪烁
4. 如果全部可达，弹出成功提示

### AMCL 粒子云可视化

**位置：** 3D 场景中（Relocate 模式下可见）

- 显示 AMCL 定位粒子分布（彩色球体 + 方向锥体）
- 颜色反映定位质量：绿色=好、黄色=一般、红色=差
- Mock 模式下粒子会随时间自动漂移，模拟真实 AMCL 退化

### 亮色主题

**位置：** 状态栏 **☀** 按钮

点击切换亮色/暗色主题。支持高对比度模式。

### 国际化

**位置：** 侧边栏底部语言选择

支持三种语言：English / 中文 / 日本語

## 八、连接真实机器人（ROS 完整流程）

```bash
# 1. 启动 ROS 核心
roscore

# 2. 启动 rosbridge WebSocket 服务器
roslaunch rosbridge_server rosbridge_websocket.launch

# 3. 启动导航栈（导航功能需要）
roslaunch <your_robot> navigation.launch
```

然后在浏览器中操作：
1. 连接 WebSocket（默认 `ws://localhost:9090`）
2. 确认地图已加载（3D 场景显示真实地图）
3. 使用 Relocate 模式设置初始位姿（如需要）
4. 选择操作模式开始编辑和导航

**ROS 话题对应：**

| WebRop 功能 | ROS 话题 | 消息类型 |
|-------------|----------|----------|
| 导航目标 | `/move_base_simple/goal` | `PoseStamped` |
| 初始位姿 | `/initialpose` | `PoseWithCovarianceStamped` |
| 地图 | `/map` | `OccupancyGrid` |
| 里程计 | `/odom` | `Odometry` |
| AMCL 粒子 | `/particlecloud` | `PoseArray` |
| 导航路径 | `/move_base/NavfnROS/plan` | `Path` |
| 速度控制 | `/cmd_vel` | `Twist` |
| HRZ 区域 | `/hrz_zones` | `String` (JSON) |
| HRP 路径 | `/hrp_path` | `Path` |

## 九、常见问题

**Q: 打开页面后只看到灰色地面，没有地图？**
A: 请先进入 Mock Mode，或者连接真实 ROS 环境。Mock 模式会自动生成地图。

**Q: 点击画禁区/路径没反应？**
A: 确认 Mode Selector 已切换到对应的 HRZ/HRP 模式。Navigate 模式下无法画禁区。

**Q: 禁区多边形闭合不了？**
A: 需要至少 3 个顶点后，点击第一个（黄色）顶点才能闭合。注意鼠标点击范围要在顶点附近 0.3m 内。

**Q: 画完路径机器人不动？**
A: 在 HRP 模式下需要点击 **"Follow Path"**（Mock）或 **"Publish Path"**（ROS）才会执行。画完不会自动执行。

**Q: 可以同时画多个禁区吗？**
A: 可以。每个闭合的区域会独立保存，侧边栏列表中可以分别管理。

**Q: 路径分段速度从哪里调？**
A: 侧边栏 Action Panel 的 HRP 模式下，每个路径段都有一个对应的速度滑块。

**Q: 小地图不显示？**
A: 确认已有地图数据（Mock 模式或 ROS 连接）。小地图固定在右上角。

**Q: 地图编辑的撤销和区域/路径的撤销是独立的吗？**
A: 是的。地图编辑模式下 Ctrl+Z/Y 只撤销地图操作；其他模式下只撤销区域/路径操作。两者互不影响。

**Q: 定位质量一直显示红色怎么办？**
A: 切换到 Relocate 模式，在地图上点击并拖拽重新设置机器人位姿。定位质量会恢复为绿色。

**Q: 如何给任务链中的航点设置位置？**
A: 在 Tasks 模式下，展开任务链中的 waypoint 步骤，点击 X/Z 输入框旁的坐标手动修改，或直接在地图上点击设置位置。

**Q: 多台机器人如何控制？**
A: 在 Fleet 区域点击机器人名称切换激活机器人，后续所有导航/航点操作针对激活的机器人。
